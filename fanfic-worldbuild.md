## 同人小说世界观构建、设定扩展与叙事落点

为同人或原创小说构建自洽的世界观，包括势力、魔法/科技体系、地理、历史、制度、日常生活和术语体系。目标不是堆设定，而是让设定稳定地支撑剧情、关系和主题。

## 系列导航

- 何时使用：你需要补设定、做AU、缝canon空白，或发现剧情被设定矛盾卡住。
- 上一步：`fanfic-outline`、`fanfic-character`都可以先行。
- 下一步：`fanfic-relationship`、`fanfic-write`、`fanfic-improve`。
- 切到`fanfic-literary`：如果设定的主要任务不是机制自洽，而是象征、寓言、结构支撑或意识场。
- 连续性检查：如果稿子已经写出来，需要做连续性核对而不是新建设定，回`fanfic-improve`。

## 套件共识

- **先判设定立场。** 严格canon、canon+扩展、AU、crossover的容错率完全不同。
- **设定必须能落地。** 任何新设定都要回答：它怎么改变人物选择、资源限制、冲突生成和日常生活。
- **设定也分硬软。** 不是每条都要成为铁律，但每条都要知道自己有多硬。
- **术语统一比灵感炫耀更重要。** 今天“灵网”明天“心网”会直接伤害可读性。

## 输入

- 基础世界观（作品名 / 游戏名 / 原创）
- 作者已有二创 / 原创设定
- 需要扩展的方向（势力、魔法体系、科技树、地理、历史、宗教、经济、日常生活等）
- 设定立场（严格canon / canon+扩展 / AU / crossover）
- 可选：当前大纲、现有术语表、需要解释的剧情卡点

## 先判硬度

- **严格canon：** 只能补缝，不能改底层规则。
- **canon+扩展：** 可以在空白处合理推演，但要标明“原作明确”“合理推演”“纯原创补丁”。
- **AU：** 可以改框架，但要写清哪些核心规则仍保留，哪些已经重写。
- **crossover：** 先列来源归属，不同宇宙的设定不能偷混成一套默认逻辑。

## canon 查证规则

- 默认优先级：原作正文 / 游戏内codex / 官方资料 > 作者访谈或官方设定补充 > 社区wiki。
- 找得到官方译名就用官方译名；找不到时自己定一个，但要在术语表写死。
- 来源互相打架时，不要假装没看见，直接标“版本冲突”。
- 不确定就写“不确定”，不要编。

## 方法工具箱

- **缝隙发现法：** 找原作没明说但剧情一定会碰到的空白。
- **影响扩散法：** 新设定不只写它本身，还要写它如何影响政治、经济、语言、习俗和冲突。
- **成本 / 限制法：** 每条强设定最好配成本、风险、资源约束或社会后果。
- **叙事落点法：** 每条设定都要回答“能写成什么场景”“会卡住谁”“会救谁”。
- **变更日志法：** 把新增和改动都记下来，防止后文设定漂移。

## 类型矩阵

| 类型 | 世界构建重点 |
|-----|-------------|
| 西幻 | 魔法体系、种族秩序、封建 / 教会 / 王权结构、地理气候 |
| 玄幻 | 势力层级、资源体系、修炼硬度、世界尺度 |
| 仙侠 | 境界、门派、法器、因果、天道规则 |
| 科幻 | 技术硬度、社会后果、资源与能源、信息传播方式 |
| 赛博朋克 | 公司、帮派、义体、黑市、街头俚语、阶层断裂 |
| 太空歌剧 | 帝国 / 联邦结构、外交辞令、物种差异、尺度切换 |
| 古言 / 架空 | 礼制、婚姻、宗法、官制、地域差异 |
| 都市 / 现耽 | 行业生态、圈层语言、城市空间、平台文化 |

## 构建流程

1. **原作校验。** 确认相关canon内容、译名和版本来源。
2. **缝隙发现。** 找原作没明说但剧情一定会踩到的空白。
3. **选方法。** 缝隙发现、影响扩散、成本限制、叙事落点四法至少选一主法。
4. **扩展构建。** 在原作框架内填细节，或在AU框架里重写规则。
5. **术语表。** 把人名、地名、势力名、器物名、制度名统一下来。
6. **叙事落点。** 写清这个设定能制造哪些场景、哪些资源限制、哪些关系压力。
7. **变更日志。** 每次新增或修改设定，都记一条：改了什么、为什么改、影响到哪里。
8. **冲突风险检查。** 看扩展设定之间、扩展与canon、扩展与现有大纲是否矛盾。

## 边界

- 扩展设定不能悄悄否定canon。要改，就明确写AU。
- 不要只写大词。设定最终要能落到人物行为、资源限制和剧情选择上。
- 术语一旦定下，后文统一，不要今天“灵网”明天“心网”。
- 设定 bible 是后台工具，不是正文倾倒口。该记下来，但不必全部倒进小说里。
