## 同人小说世界观构建与设定扩展

为同人或原创小说构建自洽的世界观，包括势力、魔法/科技体系、地理、历史、制度、日常生活和术语体系。目标不是堆设定，而是让设定能稳定地支撑剧情。

## 系列导航

- 何时使用：你需要补设定、做 AU、缝 canon 空白，或发现剧情已经被设定矛盾卡住。
- 上一步：`fanfic-outline`、`fanfic-character`都可以先行。
- 下一步：`fanfic-relationship`、`fanfic-write`、`fanfic-improve`。
- 切到`fanfic-literary`：如果设定的主要任务不是机制自洽，而是象征、寓言或叙事结构支撑。
- 连续性检查：如果稿子已经写出来，需要做连续性核对而不是新建设定，回`fanfic-improve`。

## 输入

- 基础世界观（作品名 / 游戏名 / “原创”）
- 作者已有的二创/原创设定
- 需要扩展的方向（势力、魔法体系、科技树、地理、种族、历史事件、宗教、经济、日常生活）
- 设定立场（严格 canon / canon+扩展 / AU / crossover）
- 可选：当前大纲、现有术语表、需要解释的剧情卡点

## 先判硬度

- **严格 canon：** 只能补缝，不能改底层规则。
- **canon+扩展：** 可以在空白处合理推演，但要清楚标明“原作明确”“合理推演”“纯原创补丁”。
- **AU：** 可以改框架，但要写清哪些核心规则仍保留，哪些已经重写。
- **crossover：** 先列来源归属，不同宇宙的设定绝对不能混写成一套默认规则。

## canon 查证规则

- 优先级默认是：原作正文/游戏内 codex/官方资料 > 作者访谈或官方设定补充 > 社区 wiki。
- 能找到官方译名就用官方译名；找不到时自己定一个，但要在术语表里写死。
- 来源互相打架时，不要假装没看见。直接标“版本冲突”。
- 不确定就写“不确定”，不要编。

## 中文高频类型矩阵

| 类型 | 世界构建重点 |
|-----|-------------|
| 西幻 | 魔法体系、种族秩序、封建/教会/王权结构、地理气候 |
| 玄幻 | 势力层级、资源体系、修炼硬度、世界尺度 |
| 仙侠 | 境界、门派、法器、因果、天道规则 |
| 科幻 | 技术硬度、社会后果、资源与能源、信息传播方式 |
| 赛博朋克 | 公司、帮派、义体、黑市、街头俚语、阶层断裂 |
| 太空歌剧 | 帝国/联邦结构、外交辞令、物种差异、尺度切换 |
| 古言/架空 | 礼制、婚姻、宗法、官制、地域差异 |
| 现耽/都市 | 行业生态、圈层语言、城市空间、平台文化 |

## 流程

1. **原作校验。** 先确认相关 canon 内容和译名来源。
2. **缝隙发现。** 找原作没明说、但剧情一定会碰到的空白。
3. **影响扩散。** 新设定不只写它本身，还要写它如何影响政治、经济、语言、日常生活和冲突生成。
4. **扩展构建。** 在原作框架内填细节，确保新设定不是为了临时救剧情。
5. **术语表。** 把人名、地名、势力名、器物名、制度名统一写下来。
6. **设定 bible / 变更日志。** 每次新增或修改设定，都记一条：改了什么，为什么改，影响到哪里。
7. **冲突风险检查。** 看扩展设定之间、扩展与 canon、扩展与现有大纲是否矛盾。

## 输出格式

```markdown
## 设定名称

### 设定立场
严格 canon / canon+扩展 / AU / crossover

### Canon基础
原作中已确定的设定。
来源：
- 来源1：
- 来源2：

### 扩展设定
新增内容，逐条标注：
- 与 canon 的关系：填充空白 / 合理推演 / 纯原创补丁
- 硬度：硬设定 / 软设定
- 对剧情的作用：

### 术语表
| 术语 | 统一写法 | 来源/备注 |

### 使用建议
这个设定可以怎么用在剧情里，给出2-3个场景钩子。

### 冲突风险
| 风险点 | 冲突对象 | 风险等级 | 处理建议 |

### 变更日志
| 日期 | 改动 | 原因 | 影响章节/角色 |
```

## 约束

- 扩展设定不能悄悄否定 canon。要改，就明确写 AU。
- 不要只写大词。设定最终要能落到人物行为、资源限制和剧情选择上。
- 术语一旦定下，后文统一，不要今天“灵网”明天“心网”。
- 设定 bible 是后台，不是正文。把该给作者看的都记下来，但不要把全部都倒进小说里。
