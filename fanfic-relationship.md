## 关系戏与CP推进设计

专门处理角色之间的张力工程：CP推进、宿敌拉扯、师徒失衡、家庭裂缝、群像关系分配。目标不是让角色“在一起”，而是让关系每一步变化都成立、有代价、能反过来推动剧情。

## 系列导航

- 何时使用：你知道人物是谁，但不知道他们之间怎么拉扯、怎么升温、怎么决裂、怎么回转。
- 上一步：`fanfic-character`、`fanfic-outline`、`fanfic-worldbuild`。
- 下一步：`fanfic-write`、`fanfic-improve`。
- 切到`fanfic-literary`：如果关系戏主要靠留白、象征、视角错位和不说破来成立。
- 切到`fanfic-character`：如果问题不在关系，而在单个角色根本还没长出稳定声音。

## 输入

- 关系双方或群像名单
- 关系类型（CP / 宿敌 / 师徒 / 朋友 / 家庭 / 队友 / 群像）
- 当前状态
- 目标状态
- 故事长度（短篇 / 中篇 / 长篇）
- 模式（网文 / 严肃文学 / 混合）
- 可选：禁区、雷点、结局类型、外部主线压力

## 先抓关系方程

对每段关系，先写四件事：

- A从B身上想得到什么
- B从A身上想得到什么
- 他们各自为什么不能直接开口
- 外部剧情怎样逼他们靠近、错开或撕裂

没有“想得到什么”，关系戏就会只剩空转对视。

## 常见关系类型

| 类型 | 核心问题 |
|-----|---------|
| CP | 欲望与防御如何同时存在 |
| 宿敌 | 互相理解的程度为何比朋友还深，却仍站在对立面 |
| 师徒 | 权力差、依赖、背叛、继承如何转化 |
| 家庭 | 爱与控制如何缠在一起 |
| 队友/群像 | 戏份、立场、秘密和忠诚如何分配 |

## 推进原则

- **关系必须有状态变化。** 从误判到试探，从试探到依赖，从依赖到背叛，哪怕只是半步，也得动。
- **推进靠压力，不靠宣言。** 让关系变化的最好方式不是“我喜欢你”，而是一次选择、一次缺席、一次站队、一次失手。
- **外部主线要咬住关系线。** 关系戏不能像切出去的番外。它必须改变后续行动。
- **每次靠近都要有代价。** 否则甜得发空。
- **每次决裂也要留下余温。** 否则断得太方便。

## 模式分流

- **网文：** 关系推进要能被读者感知，每章或每个关键节点最好有可复述的变化。
- **严肃文学：** 可以更含蓄，但关系张力仍要落到具体动作、沉默或选择上。
- **混合：** 明面推进不一定频繁，但暗流必须稳定存在。不能既不直说也不暗示。

## 流程

1. **定义关系起点。** 不是“陌生人”，而是“互相戒备的合作对象”“已经有旧伤的前任”这种可写状态。
2. **定义目标状态。** 在一起、决裂、和解、共犯、无法命名，都可以，但要写清。
3. **设计中间台阶。** 至少列3-5个状态变化节点。
4. **分配推动事件。** 每个节点由什么触发：共同行动、秘密暴露、误判、牺牲、站队、身体接触、权力变化。
5. **绑定外部剧情。** 写清这段关系如何影响主线决策，主线又如何反咬关系。
6. **检查失衡。** 是否一个角色始终被动、一个角色永远负责理解一切？如是，关系会扁。
7. **群像分配。** 群戏里每个角色都要有自己的诉求，不要让所有人只围着主 CP 转。

## 约束

- 不要只靠误会推进关系。误会最多是火花，不是发动机。
- 不要让 CP 吞掉人物个体目标。爱情线不是人格删除器。
- 甜和刀都要具体。不要只写“氛围很好”或“气氛很虐”。
- 群像里每个角色都要想要点什么，不要全员给主角当灯光。
