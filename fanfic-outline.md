## 同人小说大纲、结构方法与章节规划

从一个模糊前提出发，搭出可直接指导写作的剧情骨架、章节功能表、场景清单和伏笔网络。目标不是把故事写死，而是让你知道这篇文靠什么运转，以及每一章为什么存在。

## 系列导航

- 何时使用：刚有前提，还没决定主线、章节顺序、高潮位置或结构方法时。
- 上一步：无。也可以先用`fanfic-character`、`fanfic-worldbuild`补角色或设定，再回来搭骨架。
- 下一步：`fanfic-character`、`fanfic-worldbuild`、`fanfic-relationship`、`fanfic-write`。
- 成稿后：回`fanfic-improve`做返修，需要跨语种时转`fanfic-translate`。
- 切到`fanfic-literary`：如果作品的首要目标不是推进，而是语言密度、结构实验、留白、意识流或开放结尾。

## 套件共识

- **先判模式，再选结构法。** 三幕、雪花、节拍表、Story Grid都能用，不要只信一种神话。
- **结构是支撑，不是牢笼。** 大纲服务写作，不是拿来惩罚偏离。
- **章节不是全都要钩。** 网文要管理期待，文学稿要管理回声和余波；两者都不等于每章都 cliffhanger。
- **双线思维。** 主线、关系线、设定线、主题线最好同时有去处，哪怕某一章只主打一条。

## 输入

- 核心概念或“如果……会怎样”的前提
- 世界观与类型（赛博朋克 / 群星 / 西幻 / 玄幻 / 仙侠 / 科幻 / 轻小说 / 古言 / 都市 / 原创 / 其他）
- 主要角色
- 核心关系线
- 预期长度（短篇 / 中篇 / 长篇）
- 基调（黑色幽默 / 日常 / 政治惊悚 / 史诗 / 荒诞 / 混合）
- 模式（网文 / 严肃文学 / 混合）

## 模式分流

- **网文：** 结构清晰，章节之间要有推进感、压力量或期待点。
- **严肃文学：** 可以不按强钩子运行，但必须知道停顿、绕行、模糊、空白分别在服务什么。
- **混合：** 先保结构可写，再把高密度语言、实验段落集中在关键章节。不要让文学腔稀释推进，也不要让推进焦虑抹掉有价值的停驻。

## 可选结构方法

至少选一种主方法，其他方法当校验工具：

- **三幕 / 四段法：** 适合大多数故事，抓起点、压力升级、中点变化、终局条件。
- **雪花法：** 从一句话前提扩成一段，再扩成角色弧线和场景列表。适合从模糊点子长成可写大纲。
- **节拍表：** 适合网文、商业结构和高推进文本。用关键节拍管理节奏和反转。
- **Story Grid视角：** 看类型承诺、必备场面、场景价值转折。适合返看故事为什么“有事发生却不动人”。
- **场景清单法：** 按场景列POV、目标、冲突、转值和结果。适合长篇和群像。

## 搭纲流程

1. **前提拆解。** 回答：谁想要什么，为什么现在必须做，阻力是什么，失败代价是什么。
2. **双线定位。** 同时写清主线冲突与关系线冲突。没有CP也要有主角与制度、世界、自我之间的张力。
3. **选择主方法。** 三幕、雪花、节拍表、Story Grid四选一为主。
4. **建立宏观结构。** 幕次、段落、里程碑、高潮、终局条件先立起来。
5. **列章节或场景功能。** 每章至少承担一项：推进主线、推进关系、埋设伏笔、回收信息、展示世界、改变角色状态。
6. **处理章末。** 网文看期待点是否成立；文学稿看余波、反讽、空白、悬置是否成立。
7. **铺伏笔网络。** 写清埋设位置、回收位置、回收后角色认知或关系如何变化。
8. **长篇分层。** 第一幕详细，后续先写里程碑与关键转折，边写边回填，不一次性规划到窒息。

## 边界

- 大纲是骨架，不是牢笼。偏离可以，但要知道自己偏离了哪条线、为什么偏离。
- 不要把每章都做成 cliffhanger。期待点可以是危险、欲望、误解、计划、情绪后劲、回声、空白。
- 章节功能重要，但标题、意象和结构实验也可以重要；不要把功能表写成警察通缉令。
- 长篇优先保证“可写”，再追求“全知全能的大纲”。
