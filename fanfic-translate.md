## 同人小说中英互译与声线迁移

把同人小说章节在中文和英文之间互译，尽量保住文风、节拍、角色声音、圈层语感和读者契约。这不是机翻，这是在另一种语言里重建效果。

## 系列导航

- 何时使用：已有成稿，需要中译英或英译中。
- 上一步：`fanfic-write`、`fanfic-improve`、`fanfic-literary`。
- 下一步：如果译文还要继续扩写，回`fanfic-write`；如果要继续润色，回`fanfic-improve`或`fanfic-literary`。
- 切到`fanfic-literary`：如果翻译对象本身以语言实验、留白、意象和结构回声为第一目标，校译阶段以文学标准为主。
- 切到`fanfic-worldbuild`：如果你发现术语、设定或称谓体系根本还没统一。

## 套件共识

- **先定读者契约，再翻。** 这版译文到底优先要顺、要狠、要快、要冷，还是要保陌生感？
- **先定术语和称谓，再动手。** 译法一章里漂三次，比局部不完美更伤文本。
- **声音高于字面。** 字面准确但声音死掉，不算成功。
- **效果优先。** 梗、meta、俚语、脏话、节拍、留白都尽量保效果，而不是迷信逐词对应。

## 输入

- 源文本
- 翻译方向（中→英 / 英→中）
- 目标读者（中文网文读者 / AO3读者 / 文学期刊读者 / 其他）
- 模式（网文 / 严肃文学 / 混合）
- 可选：官方译名、术语表、角色卡、设定文档

## 策略先行

动笔前，先在内部定四件事：

- **读者契约：** 这版译文首先要顺、要狠、要好读，还是要保陌生感？
- **术语表：** 人名、地名、势力名、义体 / 法器 / 境界 / 称谓怎么统一？
- **称谓策略：** 师尊、师兄、senpai、choom、fixer之类，是直译、意译、保留原词，还是混合处理？
- **归化 / 异化：** 哪些地方贴近目标语言习惯，哪些地方保留原文化纹理？

## 方法工具箱

- **读者契约法：** 先定义目标读者会接受怎样的节奏、语域、陌生度。
- **术语表法：** 人名、地名、势力名、称谓、专有名词先统一。
- **声线迁移法：** 检查角色说话方式、叙述语气、幽默密度、哲学味道是否被搬过去。
- **朗读检查法：** 用朗读抓翻译腔、语序僵硬、过度解释。
- **难点附记法：** 无法无损处理的梗、文化负载词、双关、术语争议，放在正文后单列处理方案。

## 翻译原则

- **节奏优先于字面准确。** 直译会打断节拍时，优先意译。
- **角色声音不变。** 粗糙的角色译出来也要粗糙，优雅的也要优雅，嘴硬的也要嘴硬。
- **默认不在正文加译注。** 真需要说明时，正文后单列处理方案。
- **保留梗。** 能找到目标文化的等效梗就找；找不到就保原味，但要让上下文兜得住。
- **官方译名优先。** 原作已有公认译法时，不要自创一套。

## 模式分流

- **网文：** 爽点、笑点、反转、章节推进优先，句子要利索。
- **严肃文学：** 允许更高语言密度和陌生感，但不要借口“文学性”把原文重写成你自己的作品。
- **混合：** 抒情段、哲学段、意象段可适度上调质感；剧情段和对话段仍要快。

## 高频难点

- **称谓系统：** 师兄 / 老师 / 阁下 / 您 / you 的礼貌等级要稳定。
- **脏话与俚语：** 找语气强度对等，不要全部翻成一种温吞口气。
- **笑点和meta：** 优先保效果，不执着字面。
- **文化负载词：** 目标读者圈里已有约定俗成译法就跟，不要硬创新。
- **长句和短句：** 不必逐句对等，但总体呼吸和节拍要对等。

## 翻译流程

1. **通读全文。** 先把整体基调和作者习惯吃透。
2. **定策略。** 写一份小术语表和称谓表。
3. **选方法。** 读者契约、术语表、声线迁移、朗读检查中至少选一种主法。
4. **第一遍通译。** 优先保证顺和节拍。
5. **第二遍校声线。** 查角色声音、术语一致、世界观准确性。
6. **第三遍朗读。** 把翻译腔、机械解释、语序僵硬的地方削掉。
7. **最后查统一性。** 同一个词、同一个人、同一种关系，不要前后漂移。

## 翻译腔排查

- 英译中是否出现“做了一个……的动作”“给予了一个眼神”这类硬直译？
- 中译英是否把本来利索的对话翻成冗长完整句？
- 角色说话方式是否被统一成标准书面语？
- 文本的粗砺感、暧昧感、轻浮感、狠劲、meta味有没有在翻译里被抹平？

## 输出

直接输出翻译后的正文。

如有无法无损处理的梗、文化特定内容、多种可行术语或称谓方案，在正文之后单列“处理说明”。
